# 🖼️ QR Code Images Setup

## Quick Setup

The 2 QR code images you provided need to be saved to the frontend public folder.

### Step 1: Save Images

Save the 2 QR code PNG files to:

```
Petconnet_FE/public/images/qr-codes/
├── qr-momo.png          (MoMo: 0834339521 - Nguyễn Hữu Giàu)
└── qr-tpbank.png        (TPBank: 02600647401 - richdesu)
```

**If the directory doesn't exist, create it manually:**
```bash
# Windows
mkdir Petconnet_FE\public\images\qr-codes

# Linux/Mac
mkdir -p Petconnet_FE/public/images/qr-codes
```

### Step 2: Copy Files

Copy your 2 QR code images into that folder:
- **Image 1** (MoMo QR) → `qr-momo.png`
- **Image 2** (TPBank QR) → `qr-tpbank.png`

### Step 3: Restart Frontend

```bash
cd Petconnet_FE
npm run dev
```

---

## How It Works

1. Backend config (`Petconnet_BE/config/qrCodes.js`) now points to image URLs:
   - `MOMO_QR`: `/images/qr-codes/qr-momo.png`
   - `TPBANK_QR`: `/images/qr-codes/qr-tpbank.png`

2. When user creates payment, backend returns the image URL

3. Frontend displays image directly (no base64 conversion):
   ```html
   <img src="/images/qr-codes/qr-momo.png" alt="QR Code" />
   ```

4. User scans the actual QR code image

## Troubleshooting

**Q: Images not showing?**
- A: Make sure frontend is restarted (`npm run dev`)
- A: Check browser console for 404 errors on image paths

**Q: Payment page shows broken image?**
- A: Verify files exist in `public/images/qr-codes/`
- A: Check file names match exactly (case-sensitive)

**Q: Which image is which?**
- Save image with name containing "momo" or the MoMo phone number → `qr-momo.png`
- Save image with name containing "bank" or account 02600647401 → `qr-tpbank.png`
