## 📱 Testing Scenarios

### Scenario 1: User VIP Upgrade (Happy Path)

**Prerequisites**:
- Have a test user account logged in
- User is NOT yet VIP

**Steps**:
1. Open Profile page
2. Click "Nâng cấp VIP" tab
3. Select plan: "Gói năm" (Monthly 299,000đ)
4. Click "Nâng cấp ngay"
5. **Verify**:
   - ✅ Success toast appears: "Nâng cấp gói năm thành công! Chờ xác nhận từ admin"
   - ✅ Payment created in database (check MongoDB: db.payments.find({description: /VIP Subscription/}))
   - ✅ User subscriptionStatus = 'pending'

---

### Scenario 2: Admin Approves VIP Payment

**Prerequisites**:
- Logged in as Admin (adminRole = true)
- Pending VIP subscription payment exists

**Steps**:
1. Open Dashboard
2. Click "Xác nhận Thanh toán" section
3. Find subscription payment (description: "VIP Subscription - YEARLY" or "MONTHLY")
4. Click "Xác nhận" button
5. **Verify**:
   - ✅ Payment status → "completed"
   - ✅ adminApprovalStatus → "approved"
   - ✅ Success notification appears
   - ✅ User in database:
     - isPremium = true
     - subscriptionStatus = 'active'
     - subscriptionTier = 'yearly' or 'monthly'
     - subscriptionEndDate = current date + 30/365 days

---

### Scenario 3: User Sees VIP Status

**Prerequisites**:
- User has been approved for VIP (Scenario 1 + 2 completed)

**Steps**:
1. User logs out and logs back in
2. User opens Profile page
3. **Verify**:
   - ✅ "Trạng thái VIP" section shows "Đang sử dụng"
   - ✅ Subscription tier shows (monthly/yearly)
   - ✅ Expiration date displays
   - ✅ VIP badge appears on profile

---

### Scenario 4: User Cancels Subscription

**Prerequisites**:
- User is active VIP member

**Steps**:
1. User opens Profile → VIP section
2. Clicks "Hủy đăng ký"
3. Confirms cancellation
4. **Verify**:
   - ✅ subscriptionStatus = 'cancelled'
   - ✅ User still sees VIP until subscription end date
   - ✅ After end date: VIP benefits expire
   - ✅ Success message: "Đã hủy đăng ký VIP"

---

### Scenario 5: Admin Rejects VIP Payment

**Prerequisites**:
- Pending VIP subscription payment exists

**Steps**:
1. Admin opens Dashboard → "Xác nhận Thanh toán"
2. Finds subscription payment
3. Clicks "Từ chối"
4. Types rejection reason: "Lý do từ chối thanh toán"
5. Confirms
6. **Verify**:
   - ✅ Payment status → "failed"
   - ✅ adminApprovalStatus → "rejected"
   - ✅ rejectionReason stored in database
   - ✅ User remains NOT VIP
   - ✅ Toast shows success

## 🔍 Database Verification

### Check Payment Created
```javascript
// In MongoDB Console
db.payments.findOne({description: /VIP Subscription/})
// Expected response:
{
  userId: ObjectId("..."),
```

## 🐛 Common Issues & Solutions

| Issue | Cause | Solution |
|-------|-------|----------|
| User doesn't see upgrade button | Component not loading | Check browser console for errors |
| Payment not created | API endpoint error | Check backend logs for POST /subscriptions/upgrade errors |
| Admin doesn't see payment | Payment has wrong status | Check Payment.adminApprovalStatus = 'pending' |
| VIP not activated after approval | Frontend not refreshed | User needs to logout/login or refresh page |
| Subscription service import error | Wrong import path | Verify imports in subscriptionService.ts are: `./apiClient` and `../types` |
| TypeScript compilation fails | Missing types | Run: `npm run build` to see detailed errors |
